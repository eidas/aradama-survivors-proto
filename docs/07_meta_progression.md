# メタ進行設計: 鍛錬(ノロ持ち帰り→恒久強化)

> 方向性①「ラン終了時にノロを恒久通貨として持ち帰り、初期ステータスを買う」の設計書。
> 実装タスクは BACKLOG の M1〜M3。数値は初期案であり `src/data/meta.ts` に集約して調整可能にする。

## 1. コンセプトと原作設定上の位置づけ

- 倒した荒魂は不定形の**ノロ**に戻る。刀剣類管理局はノロを回収・管理している — この「回収」をメタ進行の器にする。
- ラン終了(勝敗問わず)時、稼いだノロの一部を「回収量」として持ち帰り、**鍛錬**(トレーニング)に注ぎ込んで刀使の基礎力を恒久強化する。
- 名称は「鍛錬」(タイトル画面のメニュー名)。強化はキャラ共通(部隊全体の練度が上がるイメージ)。

## 2. 経済設計

### 2.1 持ち帰りノロ(収入)

```
持ち帰りノロ = floor(ラン中の取得XP合計 × 0.10) + 勝利ボーナス 100
```

- 「取得XP合計」はレベルアップで消費される前の累計(GameScene で加算時に別途記録)
- 目安収入(docs/05 の計測より): 序盤敗北(~4分)で 15〜30、中盤敗北(~8分)で 60〜100、完走勝利で 250〜400
- 敗北でも必ず前進する(サバイバーライクの定石)。ただし放置稼ぎを防ぐため撃破数 0 のランは 0

### 2.2 鍛錬メニュー(支出)

キャラ共通・5 系統 × 各 5 段階。コストは段階ごとに逓増: `cost(n) = 50 × n`(n=1..5、1系統の総額 750)。

| 系統 | 1段階あたりの効果 | 最大(5段階) |
|---|---|---|
| 斬撃の鍛錬 | 基礎攻撃力 +2% | +10% |
| 体力の鍛錬 | 最大HP +3% | +15% |
| 脚さばき | 移動速度 +1.5% | +7.5% |
| 写しの練度 | 写し容量 +3% | +15% |
| 回収の心得 | ノロ吸引半径 +4% と 持ち帰りノロ +4% | +20% |

- 全取り総額 3,750 ≒ 勝利 10〜15 回ぶん。完走前のプレイヤーは敗北ランの積み重ねで 1〜2 系統を伸ばせる
- **上限を意図的に低くする**(最大でも 1 割強)— ラン内ビルドが主役で、メタ進行は「壁をわずかに削る」補助。パワーインフレでラン設計(docs/03)を壊さない
- リセット(振り直し)は返金 100% で提供(実験を推奨する設計)

### 2.3 効果の適用

- 効果は乗算修飾として **ラン開始時に一度だけ** Player の実効値へ合成する(RunMods と同じ流儀で `metaMods` を別枠にし、ラン内強化の乗算と干渉させない)
- リザルトに「持ち帰りノロ +N(累計 M)」を表示

## 3. セーブ拡張

```ts
// v1 のまま後方互換フィールドを追加(欠損時は 0 で補完 — 既存の loadSave マージ方式)
interface SaveData {
  best: BestRecord | null;
  settings: { muted: boolean; damageNumbers: boolean };
  noro: number;                        // 所持ノロ
  training: Record<TrainingId, number>; // 系統ID → 取得段階(0..5)
}
```

## 4. UI

- タイトル画面に「鍛錬」を追加(キー T かクリック) → TrainingScene
- TrainingScene: 5 系統を縦リスト表示(名称/現段階●●●○○/次段階の効果/コスト)。所持ノロを右上に常時表示。
  選択で購入、R でリセット(確認つき)、ESC で戻る。カード UI は LevelUpScene の意匠を踏襲
- CharacterSelect の各カードに鍛錬適用後の実効値を表示(任意。M3 のスコープ判断に委ねる)

## 5. 実装タスク分割(Gate 0 適用済み)

| ID | 内容 | 主なファイル |
|---|---|---|
| M1 | データ+純ロジック+セーブ拡張: `src/data/meta.ts`(系統定義・コスト曲線・持ち帰り式)、save.ts 拡張、購入/リセット/持ち帰り計算の純関数+Vitest | data/meta.ts, core/save.ts, core/meta.ts, tests/ |
| M2 | 鍛錬UI: TrainingScene(表示・購入・リセット・戻る)、タイトルに入口追加 | scenes/TrainingScene.ts, TitleScene.ts, main.ts |
| M3 | ラン組み込み: 取得XP累計の記録、endRun で持ち帰り加算、リザルト表示、metaMods のラン開始時適用 | GameScene.ts, Player.ts, ResultScene.ts |

- 依存: M1 → M2/M3(M2 と M3 は並行可能だが、ループ運用上は直列で回す)
- 各タスクとも受け入れ条件は BACKLOG 側に転記して管理する

## 6. リスクと対応

| リスク | 対応 |
|---|---|
| メタ強化でバランス計測(docs/05)の前提が崩れる | balance-sim は鍛錬 0 段階のセーブで実行する規約を docs/05 に追記(M3 で対応) |
| 全取り後のノロが死蔵になる | 当面は許容。将来「ノロ奉納(コスメ/実績)」を LATER に積む |
| インフレ要望への圧力 | 上限 +10〜20% を設計原則として docs 07 に明記(本書 §2.2)。変更する場合は本書の改訂を必須にする |
